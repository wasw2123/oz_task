# 하드웨어 구조

1 CPU
ALU: 산술연살논리 장치
CU: 컨트롤 유닛
Register Set: 임시 저장 장치

2 메인 메모리: 실행한 파일이 저장되는 위치

3 버스 시스템: 데이터를 주고 받기 위한 시스템

4 보조 기억 장치: SSD, HDD 등 저장한 데이터가 위치

5 그외 IO장치: 마우스, 키보드, 프린터, 디스플레이 등

ALU 

#### CPU
`Arithmetic Logic Unit` 산술 논리 연산 장치
산술: + - * / 사칙연산
논리: and or not 크기 비교
레지스터에서 데이터를 가져와서 계산후 다시 레지스터로 보냄

`Control Unit` 제어 장치
명령어를 해석하고 신호를 보내 프로그램이 순서대로 실행되도록 전체적인 스케줄 관리
(ALU에게 어떤걸 어떻게 해야하는지 보내는 역할)

`Register` 임시 저장 장치
계산이 필요한 데이터나 명령어 주소를 잠깐 저장하는 장소


#### 주, 보조 메모리 차이
주 메모리: 전원 종료했을 떄 데이터 사라짐, 속도 빠름, 프로그램을 실행했을 때 해당 데이터를 갖고 있음
보조 메모리: 전원 종료해도 데이터 유지, 가격이 저렴, 평상시에 데이터를 저장하고 있음

#### Bus System
데이터: 데이터가 오가는 통로, 데이터나 연산 결과
어드레스: 데이터가 어디에서 어디로 이동하는지
컨트롤: 때, 언제 읽고 써야하는지

# CPU의 이해
위 내용 참고 후 추가 내용
클럭 1초에 명령어를 수행할 수 있는 횟수 
1GHz = 1초당 약 1억회 정도의 작업을 수행할 수 있다.
프로세스는 1번에 1개만 수행할 수 있으나 매우 빠르니 멀티와 다를 게 없이 느껴짐
작업의 크기: CPU에 표시되는 32비트 64비트에 따라 -> 현재는 64로 생상되는 추세니 64로
64개(8바이트)의 0과 1의 기계어를 읽기, 해석, 수행, 저장까지 하는 것

# 프로그램 실행 과정
`폰 노이만 구조`
프로그램이 실행되면 메모리에 올라간 후 CPU와 데이터를 주고 받으며 돌아가게 된다

프로그램 코드 -> 어셈블리 -> 바이너리 0과 1로만 이루어짐
실행 -> 메모리 올라가며 -> 메모리에서 cpu로 이동, 해석, 수행
버스 시스템을 통해 이동 버스는 위 내용 참고

# 인터럽트
CPU에 현재 진행중인 작업을 방해하는 신호
*우선적으로 처리할 작업이 발생했다.
*정상적으로 수행할 수 없는 명령어가 입력됐다
exception(예외)

현재 실행중인 작업을 중지하고 인터럽트를 수행하고 다시 작업 수행
