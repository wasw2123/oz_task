### 1. shell script 무엇인가
확장자가 .sh로 된 파일로 터미널에서 사용하던 명령어를 그대로 사용   
echo "": python print()와 동일하게 터미널에 출력해줌        
mypy . (poetry run mypy .): mypy를 전체디렉토리 대상으로 실행       
sh확장자 파일에 터미널에서 사용했던 명령어를 나열후 실행하면 순차 실행    

### 2. Github actions
올리거나 내려받을 때 yaml확장자 파일내에 정의된 프로세스대로 동작함     
이번에는 올렸을때 (push) 자동적으로 우분투에서 파이썬, 포이트리, 블랙, 러프, 마이파이를         
세팅하고 코드를 점검함      

### 3. Github action script
작동 - 작동 환경을 설정 - 우분투, 실행할 라이브러리 설치 및 실행 명령어 작성        
name은 해당 작동 내용 - 우리가(개발자가) 읽기 편하게 작성요     
run 하고자하는 명령어 - 터미널에서 사용한 것과 같이     
그외 더 있으나 필요에 따라 문서를 참고하면 좋을 것으로 판단된다.        

### 4. Cache
깃허브 액션을 사용할 때마다 환경을 구축하면 추후 많은 시간이 소요되는 것을 알 수 있다.      
비공개 프로젝트의 경우 추후 유료로도 전환된다고 한다.       
캐시를 사용한다면 시간, 돈 양쪽 비용이 모두 절감할 수 있어 큰 이득이 된다       

다만 이번에 세팅해봤으나 정상작동이 불가하여 추후 공부하여 다시 도전해보려 한다.        

캐시는 리포지토리당 10기가 제한, 7일내 사용이 안된 경우 삭제        
동일 키를 가졌는지 체크하고 있으면 불러와서 수행 없을 경우 새로 설치하여 진행
