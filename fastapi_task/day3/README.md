### 1. SPEC API 란 무엇인지 이해하고 활용할 수 있다.
작성한 api의 설명서 작동을 테스트해볼 수도 있다.
fastapi의 경우 포함돼 있어 api를 만들기만 해도 작성된다.

### 2. open API 의 Swagger API docs 에 대해서 이해하고 활용할 수 있다.
open api는 spec api를 쉽게 작성할 수 있게 도와주는 패키지이며 협업(프론트)개발자에게 쉽게 api내용을 전달할 수 있다. 프레임워크에 포함이 안된 flask나 순수 파이썬만 써서 api작성할 때 도움을 받기 좋을 것으로 보임

### 3. 써드 파티 라이브러리 에 대해서 이해하고 있다.
나와 내가쓰는 언어에 자체내장된 라이브러리를 제외하고 poetry add xxx 이나 pip install xxx 으로 받아서 사용해야하는 라이브러리 pandas bs4 등등등 특정기능을 사용하거나 도움을 받기 위해 사용

### 4. 암호화, 복호화에 대해서 이해하고 활용할 수 있다.
암호화 - 데이터를 무작위로 보이게 변경하는 것       
복호화 - 무작위였던 데이터를 다시 원래대로 돌리는 것

아직 해당부분은 더 봐야할 내용이 있는 것으로 판단

### 5. Sqids 파이썬 라이브러리에 대해서 이해하고 활용할 수 있다.
숫자를 입력하면 랜덤한 것 같은 문자열로 바꿔주는 라이브러리
리스트로 된 숫자를 받아 하나의 문자열로 만들어 주는 것이 가장 큰 특징
단 같은 값을 넣으면 같은 값을 출력하므로 암호로 사용할 순 없음


### 6. FastAPI 의 내부에 구현된 모듈, 함수를 이용하여 SPEC API 에 대한 라우터를 구성할 줄 안다.
pydantic을 으로 schema 구성     
연산, 검출, 데이터 가공(대소문자 소량 변경이나 문자열로 변경등) model, field_validator, computed_field로 커스텀해서 구성
